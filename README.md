# yoyo-news

Daily news digests powered by an LLM (Groq). The model does both gathering and summarization — no separate search API. No login; config stored locally; history stored in DynamoDB.

## Structure

- **`docs/`** — [REQUIREMENTS.md](docs/REQUIREMENTS.md), [TECH_SPEC.md](docs/TECH_SPEC.md)
- **`frontend/`** — React (Vite) app; config (countries, language) in localStorage; fetches digests from API
- **`backend/`** — Node.js Lambdas: daily job (LLM → DynamoDB) and read API (GET /digests, GET /digests/:id, POST /digests/refresh)
- **`infrastructure/`** — AWS CDK: DynamoDB, Lambda, API Gateway, EventBridge, S3

## Prerequisites

- Node.js 20+
- AWS CLI configured (default region for deploy: **ca-central-1**)
- (Optional) [Groq](https://console.groq.com) API key for real digests. Set **GROQ_API_KEY** or **GROK_API_KEY** on the **DailyJob** Lambda (scheduled run) and on the **Read API** Lambda (for the in-app “Digest” manual refresh). Without it, the job uses mock data.

## Quick start

The **backend is deployed to AWS** (Lambda, API Gateway, DynamoDB). The frontend always talks to that deployed API — both when you run `npm run dev` locally and when you use the production build.

### 1. Install dependencies

```bash
cd backend && npm install && cd ..
cd frontend && npm install && cd ..
cd infrastructure && npm install && cd ..
```

### 2. Deploy infrastructure (backend)

AWS services default to **ca-central-1**. Override with `CDK_DEFAULT_REGION` if needed.

```bash
cd infrastructure
npx cdk bootstrap   # once per account/region (ca-central-1)
npx cdk deploy
```

Note the **ApiUrl** in the deploy output (e.g. `https://xxxxxxxxxx.execute-api.ca-central-1.amazonaws.com`).

**Real digests (no more mock data)**  
The daily job uses **mock** data until you set a **Groq** API key on the **DailyJob** Lambda:

1. **AWS Console** → **Lambda** (region **ca-central-1**) → open **YoyoNewsStack-DailyJob...**
2. **Configuration** → **Environment variables** → **Edit** → add:
   - **GROQ_API_KEY** or **GROK_API_KEY** — [Groq API key](https://console.groq.com/keys)
   - **GROQ_MODEL** — (optional) e.g. `llama-3.1-8b-instant` (default) or `llama-3.3-70b-versatile`
3. **Save**, then run the daily job again (Test in Lambda, or `npm run invoke-daily`). New digests are generated by Groq (gather + summarize in one call).

For the in-app **“Digest”** button (manual refresh of today’s news), set the same **GROQ_API_KEY** (and optionally **GROQ_MODEL**) on the **YoyoNewsStack-ReadApi...** Lambda.

**Note:** If you run `cdk deploy` again, re-add these env vars in the Console if they are cleared.

### 3. Point the frontend at the deployed API

Create `frontend/.env` (or set env when running/building):

```bash
# Use the ApiUrl from step 2 — same for local dev and production
VITE_API_URL=https://xxxxxxxxxx.execute-api.ca-central-1.amazonaws.com
```

### 4. Run frontend locally (dev)

```bash
cd frontend
npm run dev
```

Open http://localhost:5173. The app uses the deployed API from `VITE_API_URL`.

### 5. Build frontend for production

```bash
cd frontend
# VITE_API_URL from .env is used when building
npm run build
```

Deploy the `frontend/dist` output to your S3 bucket (or the CDK stack will do it if you run `cdk deploy` again after building).

## API

- **GET /digests?from=YYYY-MM-DD&to=YYYY-MM-DD** — list digest summaries
- **GET /digests/:id** — get one digest by date (id = YYYY-MM-DD)
- **POST /digests/refresh** — generate and save today’s digest (used by the in-app “Digest” button)

CORS is enabled for all origins. No auth.

## CI/CD (GitHub Actions)

- **CI** (`.github/workflows/ci.yml`): On every push and PR to `main`/`master` — installs deps, builds backend and frontend, runs `cdk synth`. No secrets required.
- **Deploy** (`.github/workflows/deploy.yml`): On push to `main` (or **Run workflow** manually) — builds and runs `cdk deploy`. Requires GitHub **Secrets**:
  - **AWS_ACCESS_KEY_ID** — IAM user access key with permission to deploy the CDK stack (and bootstrap).
  - **AWS_SECRET_ACCESS_KEY** — IAM user secret key.
  - **VITE_API_URL** — Your deployed API URL (e.g. from CDK output `ApiUrl`). Used when building the frontend so the deployed app calls the right API. Set this after the first deploy.

To use **master** as the deploy branch, edit `deploy.yml` and change `branches: [main]` to `branches: [master]`.

## Schedule

The daily job runs at **06:00 UTC** (EventBridge rule). To trigger it manually from the repo:

```bash
cd infrastructure
npm run invoke-daily
```

Or invoke the `DailyJob` Lambda from the AWS console or CLI.

## Viewing CloudWatch logs (e.g. 500 on /digests/refresh)

The **Read API** Lambda (handles GET /digests, GET /digests/:id, POST /digests/refresh) writes errors to CloudWatch. To see why a request fails:

**Option 1 – Script**

```powershell
cd infrastructure
npm run view-logs       # last hour of logs
npm run view-logs:tail  # stream logs live (then trigger "Digest" in the app)
```

Or from `infrastructure/scripts`: `.\view-readapi-logs.ps1` / `.\view-readapi-logs.ps1 -Tail`

**Option 2 – AWS Console**

1. Open **AWS Console** → **CloudWatch** (region **ca-central-1**).
2. **Logs** → **Log groups**.
3. Open the group **`/aws/lambda/YoyoNewsStack-ReadApi...`** (name includes a random suffix from CDK).
4. Open the latest **Log stream** and look for `POST /digests/refresh failed:` or other errors.

**Option 3 – AWS CLI**

```bash
# List log groups to get the exact ReadApi name
aws logs describe-log-groups --region ca-central-1 --log-group-name-prefix /aws/lambda/YoyoNewsStack-ReadApi

# Tail logs (replace LOG_GROUP_NAME with the name from above)
aws logs tail /aws/lambda/LOG_GROUP_NAME --region ca-central-1 --follow --format short
```

Trigger the failing request (e.g. click “Digest” in the app), then check the new log stream or the tail output for the error message and stack trace.

### If refresh times out after 3 seconds

The Read API Lambda may still have the default 3s timeout. Set it to 60s once:

```powershell
cd infrastructure
npm run set-readapi-timeout
```

Or in **AWS Console** → **Lambda** → open **YoyoNewsStack-ReadApi...** → **Configuration** → **General configuration** → **Edit** → **Timeout** 1 min → **Save**.
